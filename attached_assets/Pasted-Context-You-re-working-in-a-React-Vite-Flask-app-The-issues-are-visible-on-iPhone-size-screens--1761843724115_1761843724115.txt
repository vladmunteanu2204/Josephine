2) Hiking Stats bar doesn’t show during tracking

Root cause to fix:
The component is likely rendered only when distance > 0 or after GPS points exist. It must render when isTracking === true.

Tracking.jsx – render logic
Find where the stats are conditionally rendered. Replace any condition like:

{stats.distance > 0 && <HikeStats .../>}


with:

{isTracking && <HikeStats stats={stats} progress={progress} />}

3) Remove unwanted gap between header and hero

Cause: margin collapsing / inconsistent page paddings under a sticky header.

index.css (global) – standardize header + page top padding

:root { --header-h: 64px; }

/* Sticky header */
header {
  position: sticky;
  top: 0;
  z-index: 1000;
  margin: 0;        /* prevent margin-collapsing artifacts */
  padding: 0;
}

/* First section under header across the app */
main, .page-wrap, .hero-section, .page-hero {
  margin-top: 0 !important;
  padding-top: calc(var(--header-h) + env(safe-area-inset-top, 0px));
}

/* Ensure the very first child doesn’t add stray margin */
main > *:first-child,
.page-wrap > *:first-child {
  margin-top: 0 !important;
}


Check Header.css
Remove/avoid overflow:hidden on the header; it can clip content and create visual artifacts with sticky:

header { overflow: visible; }


Acceptance for (3):

On Home and Trail Detail, there is no extra blank band; hero aligns cleanly under the header across breakpoints.
