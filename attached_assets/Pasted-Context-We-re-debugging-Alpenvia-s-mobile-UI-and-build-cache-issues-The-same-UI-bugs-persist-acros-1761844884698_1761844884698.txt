The same UI bugs persist across devices even after multiple edits, which strongly suggests that the old cached PWA/service worker version is still being served.

🧹 Step 1 — Full cache + SW cleanup logic

Edit the frontend service worker (web-frontend/public/sw.js) to include a versioned cache name and automatic invalidation of old versions.
Make these exact edits:

// sw.js
const SW_VERSION = 'v10'; // bump this each deployment
const STATIC_CACHE = `static-${SW_VERSION}`;

self.addEventListener('install', event => {
  self.skipWaiting();
  event.waitUntil(
    caches.open(STATIC_CACHE).then(cache => cache.addAll(['/']))
  );
});

self.addEventListener('activate', event => {
  event.waitUntil((async () => {
    const names = await caches.keys();
    await Promise.all(
      names.filter(n => !n.includes(SW_VERSION)).map(n => caches.delete(n))
    );
    await self.clients.claim();
  })());
});


Then ensure no stale SW stays in cache.

⚙️ Step 2 — Force immediate SW update + reload

In web-frontend/src/main.jsx or index.jsx, ensure the SW registration triggers a reload on version change:

if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js').then(reg => {
    reg.update(); // check for new SW on load
    navigator.serviceWorker.addEventListener('controllerchange', () => {
      window.location.reload();
    });
  });
}

🧾 Step 3 — Prevent caching of sw.js and index.html

In the Flask backend (backend/app.py), add this logic:

from flask import make_response, send_from_directory

@app.route('/sw.js')
def service_worker():
    resp = make_response(send_from_directory('../web-frontend/dist', 'sw.js'))
    resp.headers['Cache-Control'] = 'no-store, no-cache, must-revalidate, max-age=0'
    return resp

@app.after_request
def add_cache_headers(resp):
    if resp.mimetype == 'text/html':
        resp.headers['Cache-Control'] = 'no-store, no-cache, must-revalidate, max-age=0'
    return resp


✅ This ensures browsers never reuse stale HTML or SW files.

📱 Step 5 — Clear existing stale caches on device

Create a one-time debug button or dev-console snippet that executes this:

(async () => {
  const regs = await navigator.serviceWorker.getRegistrations();
  for (const r of regs) await r.unregister();
  if ('caches' in window) {
    const names = await caches.keys();
    await Promise.all(names.map(n => caches.delete(n)));
  }
  location.reload(true);
})();


That fully clears any residual SW or cache on Safari and Chrome.
